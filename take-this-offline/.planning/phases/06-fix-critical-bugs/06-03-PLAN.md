<objective>
Add `getUser()` to the `/archive` page so that logged-in users see the Profile nav link in `SiteHeader`.

Purpose: The v1.0 audit found that the archive page calls `<SiteHeader />` with no props. `SiteHeader` accepts an optional `user` prop and conditionally renders a Profile link when it is set. Because the archive page never fetches the session, the `user` prop is always `undefined` — meaning logged-in users browsing the archive never see the Profile link, creating inconsistent navigation. This is a one-line fetch + one-prop pass.

Output: Updated `src/app/archive/page.tsx` that fetches the Supabase user and passes it to `SiteHeader`.
</objective>
