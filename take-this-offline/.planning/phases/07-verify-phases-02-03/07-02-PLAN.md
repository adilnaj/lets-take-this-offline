**ACTV-03** — User can submit a sentence for AI feedback (login required, rate-limited).
- Check src/components/activities/sentence-activity.tsx: is there a textarea + submit? Is there an auth guard (redirect to sign-in or disabled state for anonymous users)?
- Check src/app/api/sentence-feedback/route.ts (or equivalent): does it call the Anthropic SDK and return feedback?

- Check that getDisstractors() is called (from src/lib/activities.ts) to supply the wrong answers.

**ACTV-05** — User sees a "completed today" state after finishing all activities.
- Check src/components/practice-section.tsx: is there a completion state (e.g., all activities done = show CompletionBanner or disabled state)?
- Check src/components/completion-banner.tsx: does it exist and is it rendered by PracticeSection?
- The "cannot re-trigger activities" gating must also be present: if completions are tracked and activities are conditionally disabled/hidden when already completed today.

**GAME-01** — User earns points for each activity completed.
- Check src/app/api/activity/complete/route.ts: does the POST handler award points (insert/update user_stats or user_activity with a points value)?
- Check the gamification migration (20260314000000_gamification_schema.sql): confirm user_stats or user_activity table has a points column.

**GAME-02** — User earns a daily badge after completing all activities.
- Check src/app/api/activity/complete/route.ts: is there an "all-complete bonus" that fires when all 4 activities are done?
- Check src/components/completion-banner.tsx: does the UI show a badge?

**GAME-03** — User can view their current streak counter.
- Check src/components/streak-counter.tsx: does it exist and render a streak number?
- Check src/app/word/[slug]/page.tsx or src/app/page.tsx: is StreakCounter rendered and passed streak data?
- Check src/lib/activities.ts: is there a computeStreak() function?

**GAME-04** — User can save words to favorites (login required).
- Check the gamification migration: does user_favorites (or equivalent) table exist?
- Check src/components/favorite-button.tsx: does it exist and toggle favorite state (authenticated users only)?
- Check src/app/api/favorites/route.ts: does the POST/DELETE handler exist?
- Check src/app/word/[slug]/page.tsx: is FavoriteButton rendered?
- ALL FOUR must be present for a PASS. DB schema only = GAP. Button without API = GAP.

**GAME-05** — User can view activity history across past words (login required).
- Check src/app/profile/page.tsx: does it exist and fetch the user's past word activity records?
- Check that the JSX renders a list of past words with some activity data.
- Auth gate: is the profile page protected (redirects or shows nothing for anonymous users)?

**GAME-06** — User can see a mastery rating (mastered / learning / seen) per word (login required).
- Check the gamification migration: does word_mastery (or equivalent) table exist with a mastery level column?
- Check src/app/profile/page.tsx: does it display mastery ratings alongside past words?
- Check that the ratings are user-visible in JSX (not just computed server-side but never rendered).
- Schema + API without visible UI render = GAP.

Record your findings before writing the final document.
