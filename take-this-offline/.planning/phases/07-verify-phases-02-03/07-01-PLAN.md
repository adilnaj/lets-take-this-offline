Verification criteria per requirement:

**CONT-01** — Today's word page shows header, executive summary, where/how used, usage examples.
- Check src/app/page.tsx: does it query today's word via getTodaysWord() (or equivalent)?
- Check src/components/word-card.tsx: does the JSX render title/header, exec_summary, usage_context, examples fields from the word object?
- A field that is queried but not rendered in JSX = GAP.

**CONT-02** — "Heard in the Wild" section with paraphrased citation and source link.
- Check src/components/heard-in-wild.tsx (or equivalent): does it render a citation text and an `<a>` element with href pointing to source?
- Check that this component is imported and rendered on the word page (src/app/page.tsx or src/components/word-card.tsx).

**CONT-03** — Browse the word archive (paginated list of all past words).
- Check src/app/archive/page.tsx: does it call getArchive() (or equivalent) and render a list of words?
- Check src/components/archive-list.tsx and archive-pagination.tsx: do they render the list and pagination controls?

**CONT-04** — Search the archive by keyword.
- Check src/app/archive/page.tsx: does it read a search query param and pass it to searchArchive() or getArchive()?
- Check src/components/archive-filters.tsx: does it have a text input for keyword search?

**CONT-05** — Filter the archive by category and date.
- Check src/app/archive/page.tsx: does it read category and date query params?
- Check src/components/archive-filters.tsx: does it have category chips/select AND date range controls?
- Both category AND date filter must exist for a PASS.

**PLAT-03** — Dark mode: system preference default, manual toggle.
- Check src/components/providers.tsx: is ThemeProvider (next-themes or equivalent) present with `attribute="class"` and `defaultTheme` set?
- Check src/components/theme-toggle.tsx or theme-switcher.tsx: does it allow switching between light/dark/system?
- Both system-preference default AND manual toggle must be wired for a PASS.

**PLAT-04** — Word pages are server-rendered with proper meta tags for SEO.
- Check src/app/word/[slug]/page.tsx: does it export a `generateMetadata` function that returns `title` and `description`?
- Check src/app/page.tsx: does it have metadata for the home page?
- Dynamic generateMetadata using word title/definition = PASS. Static or missing = GAP.

**PLAT-05** — Each word page has a dynamic OG image generated via Vercel OG.
- Check src/app/api/og/[slug]/route.ts: does it use ImageResponse from `next/og`?
- Check that `export const runtime = 'edge'` is present (Phase 6 fix).
- Check that generateMetadata on /word/[slug]/page.tsx references the OG route in `openGraph.images`.
- All three must be present for a PASS.

Record your findings in a working notes format before writing the final document (next task).
