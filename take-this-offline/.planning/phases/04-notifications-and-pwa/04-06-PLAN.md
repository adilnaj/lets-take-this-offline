Create `src/components/notification-settings.tsx` as a `'use client'` component.

Props interface:
```ts
interface NotificationSettingsProps {
  initialPushEnabled: boolean
  initialPushHour: number
  initialEmailEnabled: boolean
}
```

State:
- `pushEnabled: boolean` (initialized from prop)
- `pushHour: number` (initialized from prop, 0-23)
- `emailEnabled: boolean` (initialized from prop)
- `pushLoading: boolean` — disables push controls while request in flight
- `emailLoading: boolean` — disables email controls while request in flight
- `pushError: string | null` — shows inline error if permission denied or API fails

**Push toggle behavior** (when toggled ON):
1. Check `Notification.permission` — if `'denied'`, set pushError to "Push blocked in browser settings. You can still use email notifications." and reset toggle to off. Return.
2. Request permission: `const perm = await Notification.requestPermission()`. If not `'granted'`, set error and reset. Return.
3. Get SW registration: `const reg = await navigator.serviceWorker.ready`
4. Subscribe to push: `const sub = await reg.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY })`
5. Extract keys: `const auth = btoa(String.fromCharCode(...new Uint8Array(sub.getKey('auth')!)))` — base64url encode
6. POST to `/api/push/subscribe` with `{ endpoint: sub.endpoint, auth, p256dh, timezone: Intl.DateTimeFormat().resolvedOptions().timeZone, notifyHour: pushHour }`
7. On success: set `pushEnabled = true`, clear error.

**Push toggle behavior** (when toggled OFF):
1. Get existing SW push subscription: `const reg = await navigator.serviceWorker.ready; const sub = await reg.pushManager.getSubscription()`
2. If sub exists: `await sub.unsubscribe()` (removes from browser), then DELETE `/api/push/unsubscribe` with `{ endpoint: sub.endpoint }`
3. Set `pushEnabled = false`.

**Hour picker**: Native `<select>` with options 0-23 formatted as "12 AM", "1 AM" ... "12 PM" ... "11 PM". When value changes, if pushEnabled, immediately re-call the subscribe flow with the new hour (this upserts the row with updated notify_hour). If not enabled, just update state.

Helper to format hour for display:
```ts
function formatHour(h: number): string {
  if (h === 0) return '12 AM'
  if (h < 12) return `${h} AM`
  if (h === 12) return '12 PM'
  return `${h - 12} PM`
}
```

**Email toggle behavior** (ON): POST `/api/email/subscribe`. On success: `emailEnabled = true`.
**Email toggle behavior** (OFF): DELETE `/api/email/unsubscribe`. On success: `emailEnabled = false`.

Base64url encoding for VAPID keys — use this helper:
```ts
function arrayBufferToBase64Url(buffer: ArrayBuffer): string {
  return btoa(String.fromCharCode(...new Uint8Array(buffer)))
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '')
}
```
