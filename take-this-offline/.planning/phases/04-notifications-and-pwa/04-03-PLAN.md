  <name>Task 1: Install web-push and create push subscribe/unsubscribe routes</name>
  <files>src/app/api/push/subscribe/route.ts, src/app/api/push/unsubscribe/route.ts</files>
  <action>
First, install the web-push package: `npm install web-push` and `npm install --save-dev @types/web-push`

**POST /api/push/subscribe** (`src/app/api/push/subscribe/route.ts`):
- Auth: require authenticated user via `createClient()` from `@/lib/supabase/server`. If no user, return 401.
- Accept JSON body: `{ endpoint: string, auth: string, p256dh: string, timezone: string, notifyHour: number }`
- Validate: notifyHour must be 0-23 integer. Return 400 with `{ error: 'invalid_hour' }` if not.
- Upsert into `push_subscriptions` using `{ onConflict: 'user_id,endpoint' }` — update timezone and notify_hour if same device re-subscribes with new settings.
- Return 200 `{ ok: true }` on success.

**DELETE /api/push/unsubscribe** (`src/app/api/push/unsubscribe/route.ts`):
- Auth: require authenticated user. Return 401 if not.
- Accept JSON body: `{ endpoint: string }`
- Delete from `push_subscriptions` where `user_id = auth.uid() AND endpoint = body.endpoint`
- Return 200 `{ ok: true }` on success, 404 `{ error: 'not_found' }` if row did not exist (count = 0).

Both routes follow `NextResponse.json()` pattern. Both use the standard Supabase server client (RLS handles user scoping).

<task type="auto">
  <name>Task 2: Create push-dispatch cron handler</name>
  <files>src/app/api/cron/push-dispatch/route.ts</files>
  <action>
**GET /api/cron/push-dispatch** (`src/app/api/cron/push-dispatch/route.ts`):
- Security: Check `Authorization` header equals `Bearer ${process.env.CRON_SECRET}`. Return 401 if missing or wrong.
- Determine current UTC hour: `new Date().getUTCHours()`
- Fetch today's word from `words` table: select title, definition, slug where `daily_date = TODAY` (use `new Date().toISOString().slice(0, 10)` for TODAY). If no word found, return 200 `{ sent: 0 }`.
- Create a Supabase client using the service role key so it can bypass RLS:
  ```ts
  import { createClient as createServiceClient } from '@supabase/supabase-js'
  const supabase = createServiceClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!
  )
  ```
- Query all `push_subscriptions` where `notify_hour = currentHour`.
- Set up web-push VAPID details once:
  ```ts
  import webPush from 'web-push'
  webPush.setVapidDetails(
    process.env.VAPID_SUBJECT!,
    process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!,
    process.env.VAPID_PRIVATE_KEY!
  )
  ```
- For each subscription, call `webPush.sendNotification()`:
  ```ts
  const payload = JSON.stringify({
    title: word.title,
    body: word.definition.slice(0, 100),
    url: `${process.env.NEXT_PUBLIC_APP_URL}/word/${word.slug}`,
  })
  await webPush.sendNotification(
    { endpoint: sub.endpoint, keys: { auth: sub.auth, p256dh: sub.p256dh } },
    payload
  )
  ```
- If `sendNotification` throws a 410 (subscription expired/gone), delete that row from `push_subscriptions` (cleanup expired subscriptions).
- Return 200 `{ sent: N }` where N is the number of successful pushes.
- Use `export const dynamic = 'force-dynamic'` to prevent static prerender.

Do NOT write vercel.json in this plan — cron registration for both push-dispatch and email-digest is handled exclusively in Plan 04 to avoid concurrent write conflicts.
