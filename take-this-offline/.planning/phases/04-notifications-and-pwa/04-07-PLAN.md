  <how-to-verify>
Run `npm run dev` and open http://localhost:3000 in Chrome (or mobile browser).

**Verify 1 — Push notification subscription (NOTF-01):**
1. Sign in and navigate to http://localhost:3000/profile
2. Scroll to the Notifications section
3. Toggle "Daily push reminder" ON
4. Browser should show a permission dialog — click Allow
5. Confirm: toggle stays ON, no error message shown
6. Confirm: hour picker is visible and changes are accepted
7. Check Supabase Table Editor → push_subscriptions: a row exists for your user_id with the correct endpoint and notify_hour

**Verify 2 — Push unsubscribe (NOTF-03 push):**
1. Toggle "Daily push reminder" OFF
2. Confirm: toggle returns to OFF state
3. Check Supabase → push_subscriptions: the row is gone (or confirm deletion)

**Verify 3 — Push permission denied flow:**
1. In Chrome: Settings → Privacy → Notifications → Block http://localhost:3000
2. Return to /profile, toggle push ON
3. Confirm: toggle resets to OFF and message "Push blocked in browser settings. You can still use email notifications." appears below the toggle

**Verify 4 — Email digest subscription (NOTF-02):**
1. Toggle "Daily email digest" ON
2. Confirm: toggle stays ON
3. Check Supabase → email_digest_prefs: a row exists with enabled=true
4. (Optional, if Resend is configured) Manually call the cron: `curl -H "Authorization: Bearer $CRON_SECRET" http://localhost:3000/api/cron/email-digest` — check the email arrives

**Verify 5 — Email unsubscribe (NOTF-03 email):**
1. Toggle "Daily email digest" OFF
2. Check Supabase → email_digest_prefs: enabled is now false

**Verify 6 — PWA install (PLAT-01):**
1. In Chrome, open http://localhost:3000
2. In the address bar, look for the install icon (desktop install button in Chrome omnibar) OR check that SiteHeader shows an "Install" button/icon
3. Click the install prompt and install the app
4. Confirm: app opens as a standalone window (no browser chrome)
5. Confirm: the SiteHeader Install button disappears after installation

**Verify 7 — Offline today's word (PLAT-02):**
1. Visit http://localhost:3000 (today's home page) with network connected — this caches the page
2. Open Chrome DevTools → Application → Service Workers → confirm "ltto-v1" cache exists
3. Enable Offline mode in DevTools (Network tab → Offline checkbox)
4. Reload http://localhost:3000
5. Confirm: today's word page loads fully from cache with no network

**Verify 8 — Build passes:**
Run `npm run build` — confirm no TypeScript or build errors.
